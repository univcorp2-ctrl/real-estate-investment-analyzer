# インプット元データ・価格妥当性・土地値カバー率の算出仕様

## 1. 現状プロトタイプの前提

このリポジトリの現状プロトタイプは、外部データベースやAPIから自動取得していません。

現状の入力元は以下です。

1. URLクエリパラメータ
2. 画面フォームの手入力
3. 未入力項目のデフォルト値

例:

```text
?price=880&str=RC&age=0&loanAmt=880&rent=0&ir=2&ly=30&br=58&name=東村山市%20戸建て
```

この場合、以下のように解釈しています。

| 項目 | 値 | 解釈 |
|---|---:|---|
| price | 880 | 物件価格 880万円 |
| str | RC | 構造 RC |
| age | 0 | 築0年 |
| loanAmt | 880 | 借入880万円 |
| rent | 0 | 月額賃料は未入力扱い |
| br | 58 | 58,000円/月、つまり5.8万円/月の推定賃料 |
| ir | 2 | 金利2% |
| ly | 30 | 返済期間30年 |

## 3. 公的データソース候補

### 3.1 国土交通省 不動産情報ライブラリAPI

優先度が高いデータソース。

取得できる主な情報:

- 不動産取引価格情報
- 成約価格情報
- 地価公示・地価調査
- 都市計画区域・用途地域
- 将来推計人口250mメッシュ
- 駅別乗降客数
- 洪水・高潮・津波・土砂災害などの防災情報

利用にはAPI利用申請とAPIキーが必要。ブラウザから直接APIを叩くのではなく、サーバー側からAPIキー付きで取得する設計にする。

### 3.2 国税庁 財産評価基準書・路線価図・評価倍率表

土地値カバー率を出すための基礎データ。

路線価地域では、基本式は以下。

```text
土地相続税評価額 = 路線価（千円/m2） × 1,000 × 土地面積（m2） × 補正率
```

実務上は、奥行価格補正、側方路線影響加算、二方路線影響加算、不整形地補正、間口狭小補正、セットバック等の補正が必要。

路線価がない倍率地域では、基本式は以下。

```text
土地相続税評価額 = 固定資産税評価額 × 評価倍率
```

### 3.3 国土数値情報

GIS形式で使う補助データ。

用途:

- 駅・鉄道・駅別乗降客数
- 行政区域
- 用途地域
- 地価公示
- 洪水・土砂災害等のハザード
- 人口集中地区
- 将来人口メッシュ

### 3.4 民間・自社蓄積データ

公的データだけでは足りない。

必要になる民間/自社データ:

- 現在売出中の類似物件価格
- 成約に近い売買事例
- 賃貸募集事例
- 実際の成約賃料
- 管理費・修繕費の実績
- 融資承認条件
- 建物修繕履歴
- 再建築可否、接道、違法建築情報

ポータルサイトをスクレイピングする場合は、各サイトの利用規約・robots.txt・著作権・データベース権に注意する。商用運用ではAPI契約または自社入力・CSV連携を優先する。

## 4. 価格妥当性の算出仕様

価格妥当性は、現状プロトタイプでは簡易的に以下で見ている。

```text
価格妥当性 = 実際の表面利回り ÷ 期待表面利回り
```

点数化:

| 実際の表面利回り ÷ 期待表面利回り | 点数 |
|---:|---:|
| 1.30以上 | 5 |
| 1.15以上 | 4 |
| 1.00以上 | 3 |
| 0.85以上 | 2 |
| 0.70以上 | 1 |
| 0.70未満 | 0 |

現状の期待表面利回りは、構造と築年数からの仮置き。

| 条件 | 期待利回り |
|---|---:|
| RC/SRC・築20年以内 | 7.0% |
| RC/SRC・築20年超 | 8.5% |
| 鉄骨 | 8.5% |
| 木造・築20年以内 | 9.5% |
| 木造・築20年超 | 11.0% |

ただし、本番ではこの方式だけだと弱い。

本番の推奨式:

```text
類似取引価格 = 近隣売買事例から補正して算出
収益還元価格 = NOI ÷ キャップレート
積算価格 = 土地評価額 + 建物残存価値
妥当価格 = weighted_average(類似取引価格, 収益還元価格, 積算価格)
価格乖離率 = 売出価格 ÷ 妥当価格
```

推奨ウェイト例:

| 物件タイプ | 類似取引 | 収益還元 | 積算 |
|---|---:|---:|---:|
| 区分マンション | 50% | 40% | 10% |
| 戸建て賃貸 | 35% | 35% | 30% |
| 一棟アパート | 30% | 50% | 20% |
| 土地値重視の古家 | 20% | 20% | 60% |

価格妥当性の判定例:

| 価格乖離率 | 判定 | 点数 |
|---:|---|---:|
| 0.85以下 | 割安 | 5 |
| 0.85〜0.95 | やや割安 | 4 |
| 0.95〜1.05 | 妥当 | 3 |
| 1.05〜1.15 | やや割高 | 2 |
| 1.15〜1.30 | 割高 | 1 |
| 1.30超 | かなり割高 | 0 |

## 5. 類似取引価格の計算

### 5.1 取得元

- 国交省 不動産情報ライブラリAPIの不動産取引価格情報・成約価格情報
- 自社DB
- 仲介会社からの成約事例
- ポータル掲載価格の蓄積データ

### 5.2 補正式

```text
補正後単価 = 事例単価
  × 時点補正
  × 駅距離補正
  × 築年補正
  × 構造補正
  × 面積補正
  × 接道/用途地域補正
  × ハザード補正
```

```text
類似取引価格 = median(補正後単価) × 対象面積
```

最初は平均ではなく中央値を推奨。外れ値に強いため。

## 6. 収益還元価格の計算

```text
収益還元価格 = NOI ÷ キャップレート
```

キャップレートは以下から推定する。

1. 近隣の売買事例から逆算
2. 類似物件の利回り分布から推定
3. 構造・築年数・立地・ハザードで補正

例:

```text
事例キャップレート = 事例NOI ÷ 事例売買価格
対象キャップレート = median(事例キャップレート) + リスク補正
```

リスク補正例:

| 条件 | 補正 |
|---|---:|
| 駅徒歩15分超 | +0.3%〜+0.8% |
| 人口減少メッシュ | +0.3%〜+1.0% |
| ハザード該当 | +0.5%〜+1.5% |
| 築古木造 | +0.5%〜+2.0% |
| RC築浅・駅近 | -0.3%〜-1.0% |

## 7. 積算価格の計算

```text
積算価格 = 土地評価額 + 建物残存価値
```

建物残存価値の簡易式:

```text
建物再調達価格 = 構造別再調達単価 × 建物面積
残存価値率 = max(残存耐用年数 ÷ 法定耐用年数, 最低残存率)
建物残存価値 = 建物再調達価格 × 残存価値率
```

構造別再調達単価は固定値ではなく、地域・時期・建物グレードで更新する。最初のMVPでは管理画面でマスタ化する。

## 8. 土地値カバー率の算出仕様

土地値カバー率は、売買価格に対して土地評価額がどれくらいカバーしているかを見る指標。

```text
土地値カバー率 = 土地評価額 ÷ 売買価格 × 100
```

### 8.1 路線価方式

```text
土地評価額 = 路線価（千円/m2） × 1,000 × 土地面積（m2） × 補正率
```

補正率は簡易版では1.0。本番版では以下を反映する。

- 奥行価格補正
- 側方路線影響加算
- 二方路線影響加算
- 不整形地補正
- 間口狭小補正
- 奥行長大補正
- セットバック控除
- 私道負担
- 借地権割合、貸家建付地評価が必要な場合の調整

### 8.2 倍率方式

```text
土地評価額 = 固定資産税評価額 × 評価倍率
```

固定資産税評価額は自治体・納税通知書等から取得する必要がある。

### 8.3 公示地価・地価調査を使う代替式

路線価が使えない、またはAPI化が難しい場合の代替。

```text
近隣地価単価 = 距離加重平均(近隣の地価公示・地価調査地点の単価)
土地評価額 = 近隣地価単価 × 土地面積 × 取引補正率
```

注意点:

- 公示地価は標準地価格であり、対象地そのものの価格ではない
- 駅距離、用途地域、接道、容積率、形状差を補正する必要がある
- 相続税路線価は公示価格等の水準と一致しないため、土地値カバー率の定義を統一する

## 10. 実装アーキテクチャ案

```text
物件URL/手入力/CSV
  ↓
物件基本情報正規化
  ↓
住所ジオコーディング
  ↓
公的データ取得
  - 不動産情報ライブラリAPI
  - 国土数値情報
  - 国税庁路線価/評価倍率
  ↓
民間・自社データ取得
  - 賃貸事例
  - 売買事例
  - 管理費/修繕履歴
  ↓
指標計算
  - 利回り
  - NOI
  - DSCR
  - CF
  - CCR
  - 土地値カバー率
  - 価格乖離率
  ↓
6軸スコアリング
  ↓
S/A/B/C/D/E判定
```

## 11. 仕様上の重要な注意

- 現状の価格妥当性は、期待利回りとの比較による簡易版である
- 現状の土地値カバー率は、ユーザー入力された `landCoverRate` をそのまま使っている
- 土地値カバー率を自動算出するには、最低でも所在地、土地面積、路線価または固定資産税評価額が必要
- 正確な土地評価には、接道、奥行、間口、形状、私道負担、セットバック、用途地域などが必要
- 近隣賃料・売買相場は、公的データだけでは不足するため、自社DBまたは契約データが必要
