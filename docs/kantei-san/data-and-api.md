# Kantei-san データ取得・情報ソース仕様

## 0. 目的

Kantei-san の判定精度は、入力データの質に大きく依存します。

この資料では、どのデータをどこから取得し、どの計算に使うかを整理します。

## 1. データ取得の全体方針

Kantei-san では、データを3種類に分けます。

```text
1. 物件固有データ
2. 公的オープンデータ
3. 民間・自社データ
```

### 1.1 物件固有データ

物件資料や手入力から取得するデータです。

例:

- 価格
- 賃料
- 構造
- 築年数
- 土地面積
- 建物面積
- 固定資産税
- 管理費
- 修繕費
- 借入額
- 金利
- 返済期間

### 1.2 公的オープンデータ

国や自治体などが提供するデータです。

例:

- 不動産取引価格
- 成約価格
- 地価公示
- 地価調査
- 用途地域
- 将来人口
- 駅別乗降客数
- ハザード
- 人口集中地区

### 1.3 民間・自社データ

公的データだけでは足りないため、実運用では民間・自社データも必要です。

例:

- 近隣募集賃料
- 近隣成約賃料
- 管理会社の実績賃料
- 修繕履歴
- 空室期間
- 融資承認実績
- 売買成約実績

## 2. 国交省 不動産情報ライブラリAPI

Kantei-san では、国交省の不動産情報ライブラリAPIを主要な公的データ取得元として扱います。

### 2.1 使うAPI

| 用途 | API ID | 使う理由 |
|---|---|---|
| 取引価格・成約価格 | XPT001 | 価格妥当性、近隣相場 |
| 地価公示・地価調査 | XPT002 | 土地単価、積算価格、土地値代替推定 |
| 用途地域 | XKT002 | 再建築性、出口戦略、価格補正 |
| 将来人口250mメッシュ | XKT013 | 立地、賃貸需要、出口流動性 |
| 駅別乗降客数 | XKT015 | 駅力、流動性、賃貸需要 |
| 災害危険区域 | XKT016 | リスク耐性、融資・出口 |
| 大規模盛土造成地 | XKT020 | 地盤リスク |
| 洪水浸水想定区域 | XKT026 | 水害リスク |
| 津波浸水想定 | XKT028 | 水害リスク |
| 土砂災害警戒区域 | XKT029 | 土砂災害リスク |
| 人口集中地区 | XKT031 | 市街地性、出口流動性 |

### 2.2 APIキー管理

APIキーはGitHubに入れません。

管理方法:

- ローカル: 環境変数 `MLIT_REINFO_API_KEY`
- 本番: Google Cloud Secret Manager
- Cloud Run: Secret Managerから環境変数として注入

禁止:

- ソースコードに直書き
- GitHubに `.env` をコミット
- フロントエンドJSに埋め込み
- READMEやIssueに貼る

### 2.3 なぜサーバー側で呼ぶのか

ブラウザから直接APIを呼ぶと、APIキーが利用者に見えてしまいます。

そのため、以下の構成にします。

```text
ブラウザ
  ↓
Kantei-san サーバーAPI
  ↓
Secret ManagerのAPIキー
  ↓
国交省API
```

## 3. 住所と緯度経度

公的データ取得には、住所だけでなく緯度経度が必要です。

### 3.1 入力パターン

| パターン | 説明 |
|---|---|
| 緯度経度を手入力 | MVPで最も確実 |
| 住所からジオコーディング | 本番で導入 |
| 物件ポータルから取得 | 取得条件・規約に注意 |
| 登記・地番から変換 | 精度は高いが実装が重い |

### 3.2 推奨

本番では、住所正規化とジオコーディングを導入します。

必要な処理:

```text
住所入力
  ↓
住所正規化
  ↓
緯度経度取得
  ↓
町字IDや自治体コード取得
  ↓
API・GISデータ取得
```

## 4. 路線価・評価倍率

土地値カバー率を正確に出すには、路線価または固定資産税評価額が必要です。

### 4.1 路線価方式

```text
土地評価額 = 路線価 × 土地面積 × 補正率
```

必要データ:

| データ | 説明 |
|---|---|
| 路線価 | 円/m2 |
| 土地面積 | m2 |
| 奥行 | 奥行補正に必要 |
| 間口 | 間口狭小補正に必要 |
| 接道 | 側方路線、二方路線などに必要 |
| 形状 | 不整形地補正に必要 |
| セットバック | 実質利用可能面積に影響 |
| 私道負担 | 評価額に影響 |

### 4.2 倍率方式

```text
土地評価額 = 固定資産税評価額 × 評価倍率
```

必要データ:

- 固定資産税評価額
- 評価倍率

### 4.3 実装方針

国税庁の路線価図・評価倍率表について、公式の汎用機械取得APIは確認できていないため、Kantei-san では以下の順番で対応します。

1. `routeValueYenPerSqm` を手入力
2. 路線価CSVを自社DB化して参照
3. 商用APIに差し替え
4. 路線価がない場合は地価公示・地価調査で代替推定

## 5. 近隣取引価格

価格妥当性を見るには、近隣取引価格が必要です。

### 5.1 取得項目

- 取引価格総額
- 面積
- 平米単価
- 物件種類
- 構造
- 建築年
- 取引時点
- 所在地
- 対象物件からの距離

### 5.2 使い方

```text
近隣取引を取得
  ↓
対象物件からの距離で絞る
  ↓
面積、構造、築年で補正
  ↓
外れ値を除外
  ↓
中央値単価を出す
  ↓
対象面積を掛けて類似取引価格を出す
```

### 5.3 なぜ平均ではなく中央値か

不動産価格には外れ値があります。

- 極端に安い事故物件
- 極端に高い特殊物件
- 面積が小さすぎる物件
- 取引条件が特殊な物件

平均値は外れ値に引っ張られやすいので、初期版では中央値を使います。

## 6. 近隣賃料

公的データだけでは、賃料妥当性が弱いです。

### 6.1 必要なデータ

- 募集賃料
- 成約賃料
- 管理会社の実績賃料
- 空室期間
- 間取り
- 面積
- 築年数
- 駅徒歩
- 設備

### 6.2 使い方

```text
入力賃料 ÷ 近隣相場賃料
```

| 比率 | 判定 |
|---:|---|
| 0.9〜1.1 | 妥当 |
| 1.1〜1.2 | やや高い |
| 1.2超 | 高すぎる可能性 |
| 0.8未満 | 保守的、または物件に問題あり |

入力賃料が相場より高すぎる場合、収益性スコアを減点します。

## 7. ハザード情報

### 7.1 見るリスク

- 洪水
- 津波
- 土砂災害
- 災害危険区域
- 大規模盛土
- 液状化
- 高潮

### 7.2 判定方法

```text
物件の緯度経度
  ↓
ハザードポリゴンと重ねる
  ↓
該当するリスクを抽出
  ↓
リスク耐性と出口戦略を減点
```

### 7.3 表示例

```text
洪水浸水想定区域に該当しています。
想定浸水深が大きい場合、保険料、修繕費、融資、売却価格に影響する可能性があります。
```

## 8. データ品質スコア

Kantei-san では、判定結果だけでなく、データの信頼度も出します。

### 8.1 データ品質に影響するもの

| 状態 | 影響 |
|---|---|
| 賃料が未入力 | 大きく減点 |
| 土地面積が未入力 | 土地値カバー率が出せない |
| 緯度経度が未入力 | 公的API取得ができない |
| 路線価が未入力 | 土地値が推定になる |
| 近隣賃料が未取得 | 賃料妥当性が弱い |
| 近隣売買事例が少ない | 価格妥当性が弱い |
| ハザード未取得 | リスク評価が弱い |

### 8.2 表示例

```text
データ品質: 68/100
不足: 路線価、固定資産税、近隣成約賃料、修繕履歴
注意: 土地値カバー率は地価公示からの代替推定です。
```

## 11. 実務上の注意

- 公的APIのデータは完全ではありません。
- 取引価格は時期、面積、用途、取引事情で大きく変わります。
- 地価公示は対象地そのものの価格ではありません。
- 路線価は相続税評価のための値で、市場価格そのものではありません。
- 賃料相場は公的データだけでは不十分です。
- 修繕費、固定資産税、管理費を入れないと収支は甘く見えます。
- 再建築可否、接道、違法建築は必ず別途確認が必要です。
