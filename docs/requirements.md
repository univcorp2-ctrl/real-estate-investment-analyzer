# 収益物件 投資判定ロジック 要件定義・リバースエンジニアリング資料

## 1. 目的

公開画面で確認できる収益物件査定ツールの挙動をもとに、不動産投資の初期スクリーニングに使える分析ロジックを要件定義化する。

この資料は以下を目的とする。

1. どの投資基準パラメータが存在するかを整理する
2. S / A / B / C / D / E ランクの判定ロジックを推定する
3. 収益性だけでなく融資・リスク・出口まで含めた6軸評価を再現する
4. 実運用・追加検証しやすいプロトタイプ仕様に落とす

## 2. 確認できた公開情報

### 2.1 ランディングページで確認できる項目

公開ページ上では、物件一覧に以下の項目が表示されていた。

- 物件名
- ランク
- スコア
- 価格
- 利回り
- 築年数
- 構造
- DSCR
- 月間CF
- CCR
- 6軸
- 試算URL

サンプル一覧には `A 21/30`、`S 22/30`、`C 13/30`、`D 9/30` などが出ている。

重要なのは、`22/30 = S` がある一方で、`23/30` や `24/30` でも `A` の例がある点である。したがってランクは単純な総点だけではなく、危険条件による上限補正、つまりゲート判定が入っている可能性が高い。

### 2.2 Chrome Web Store 説明文で確認できる6軸

説明文から、評価軸は以下の6つと推定できる。

1. 収益性
2. 立地
3. 融資条件
4. リスク耐性
5. 価格妥当性
6. 出口戦略

また、土地値カバー率を用いて出口戦略の安全性を定量評価していると説明されている。

### 2.3 収支シミュレーターURLから読める入力パラメータ

例:

```text
/simulation?caseId=RE-20260406-12445020&price=880&str=RC&age=0&loanAmt=880&rent=0&ir=2&ly=30&br=58&name=東村山市 戸建て&email=...&ref=portfolio
```

推定される意味:

| URLパラメータ | 推定意味 | 単位 | 根拠・備考 |
|---|---:|---:|---|
| `caseId` | 案件ID | 文字列 | レポート/履歴連携用 |
| `price` | 物件価格 | 万円 | `880` は 880万円と読むのが自然 |
| `str` | 構造 | 文字列 | RC / 木造など |
| `age` | 築年数 | 年 | `0` は新築または築浅 |
| `loanAmt` | 借入額 | 万円 | 借入880万円 |
| `rent` | 月額賃料 | 万円 | 0 の場合、賃料未入力または別パラメータ使用 |
| `ir` | interest rate | % | `2` は年2% |
| `ly` | loan years | 年 | `30` は30年返済 |
| `br` | base rent / building rent | 千円 | `58` は 5.8万円/月と読むと、880万円物件の利回り約7.9%になり自然 |
| `name` | 物件名 | 文字列 | 表示用 |
| `email` | レポート送付先 | 文字列 | 実装では保存しない |
| `ref` | 流入元 | 文字列 | ナビゲーション/計測用 |

## 3. 投資分析で使う基本指標

### 3.1 表面利回り

```text
表面利回り = 年間満室賃料 ÷ 物件価格 × 100
```

初期スクリーニング用。運営費、空室、修繕、税金、融資返済を控除しない。

### 3.2 NOI

```text
NOI = 年間賃料 × (1 - 空室損率) - 年間運営費 - 固定費/修繕予備費
```

本プロトタイプでは、簡易モデルとして以下をデフォルトにする。

- 空室損率: 5%
- 運営費率: 25%
- 修繕予備費: 物件価格 × 0.5%/年

### 3.3 年間返済額

元利均等返済で計算する。

```text
月返済 = 借入元本 × 月利 × (1 + 月利)^返済回数 ÷ ((1 + 月利)^返済回数 - 1)
年間返済 = 月返済 × 12
```

### 3.4 DSCR

```text
DSCR = NOI ÷ 年間返済額
```

1.0未満なら返済を物件収入で賄えない。1.2〜1.3以上なら最低限の安全域、1.5以上なら余裕ありとして扱う。

### 3.5 月間キャッシュフロー

```text
月間CF = (NOI - 年間返済額) ÷ 12
```

### 3.6 CCR / Cash-on-Cash Return

```text
CCR = 年間税前CF ÷ 自己資金投入額 × 100
```

自己資金投入額は以下で推定する。

```text
自己資金 = max(物件価格 - 借入額, 0) + 物件価格 × 諸費用率
```

## 4. 6軸スコアリング仕様

各軸5点、合計30点満点。

### 4.1 収益性 0〜5点

主に表面利回りと月間CFで評価する。

| 条件 | 点数 |
|---|---:|
| 表面利回り12%以上 | 5 |
| 10%以上 | 4 |
| 8%以上 | 3 |
| 6%以上 | 2 |
| 4%以上 | 1 |
| 4%未満 | 0 |

補正:

- 月間CFがマイナス: -1点
- DSCRが1.0未満: -1点

### 4.2 立地 0〜5点

公開URLだけでは立地情報が不足するため、未入力時は中立値3点とする。

任意入力がある場合:

| 条件 | 加点 |
|---|---:|
| 駅徒歩5分以内 | +2.0 |
| 駅徒歩10分以内 | +1.5 |
| 駅徒歩15分以内 | +1.0 |
| 駅徒歩20分以内 | +0.5 |
| 人口増加エリア | +2.0 |
| 人口横ばい | +1.0 |
| 流動性高いエリア | +1.0 |

### 4.3 融資条件 0〜5点

DSCR、LTV、金利、返済期間で評価する。

| DSCR | 基礎点 |
|---:|---:|
| 1.50以上 | 5 |
| 1.30以上 | 4 |
| 1.15以上 | 3 |
| 1.00以上 | 2 |
| 0.90以上 | 1 |
| 0.90未満 | 0 |

補正:

- LTV100%超: -1点
- 金利4%以上: -1点
- 金利3%以上: -0.5点
- 融資年数が残存法定耐用年数を大きく超える: -0.5点

### 4.4 リスク耐性 0〜5点

損益分岐稼働率・家賃下落耐性で見る。

```text
損益分岐稼働率 = (年間返済額 + 年間運営費 + 固定費) ÷ 年間満室賃料
家賃下落耐性 = 1 - 損益分岐稼働率
```

| 損益分岐稼働率 | 点数 |
|---:|---:|
| 70%以下 | 5 |
| 80%以下 | 4 |
| 90%以下 | 3 |
| 100%以下 | 2 |
| 110%以下 | 1 |
| 110%超 | 0 |

### 4.5 価格妥当性 0〜5点

構造・築年数から期待利回りを設定し、実際の利回りと比較する。

期待表面利回りの例:

| 条件 | 期待利回り |
|---|---:|
| RC/SRC・築20年以内 | 7.0% |
| RC/SRC・築20年超 | 8.5% |
| 鉄骨 | 8.5% |
| 木造・築20年以内 | 9.5% |
| 木造・築20年超 | 11.0% |

| 実利回り ÷ 期待利回り | 点数 |
|---:|---:|
| 1.30以上 | 5 |
| 1.15以上 | 4 |
| 1.00以上 | 3 |
| 0.85以上 | 2 |
| 0.70以上 | 1 |
| 0.70未満 | 0 |

### 4.6 出口戦略 0〜5点

土地値カバー率、構造、築年数、残存耐用年数で評価する。

土地値カバー率がある場合:

| 土地値カバー率 | 点数 |
|---:|---:|
| 100%以上 | 5 |
| 80%以上 | 4 |
| 60%以上 | 3 |
| 40%以上 | 2 |
| 20%以上 | 1 |
| 20%未満 | 0 |

土地値カバー率がない場合:

| 条件 | 点数 |
|---|---:|
| RC/SRCで残存耐用年数20年以上 | 4 |
| RC/SRCで残存耐用年数10年以上 | 3 |
| 木造で築浅 | 3 |
| 法定耐用年数超過 | 1 |
| 老朽木造かつCF薄い | 0 |

## 5. 総合ランク判定

### 5.1 基本ランク

| 総点 | 基本ランク | 意味 |
|---:|---|---|
| 22〜30 | S | 積極検討 |
| 20〜21 | A | 有力候補 |
| 15〜19 | B | 条件次第で検討 |
| 12〜14 | C | 慎重検討 |
| 8〜11 | D | 原則見送り |
| 0〜7 | E | 危険/見送り |

### 5.2 ゲート判定

公開サンプルでは、総点だけでランクが決まっていない可能性が高い。そこで以下の上限補正を入れる。

| 危険条件 | ランク上限 |
|---|---|
| DSCR < 1.0 | D |
| 月間CF < 0 | D |
| 表面利回り < 5% | C |
| 土地値カバー率 < 40% かつ 法定耐用年数超過 | D |
| 家賃未入力かつ推定不能 | C |
| Sランク相当でも警告が1つ以上ある | A |

このゲート制により、例えば総点が22点以上でも、出口・融資・CFのどこかに警告がある場合は A に抑制される。

## 6. 判定結果に出すべき文言

### S

- 収益性・融資返済・出口が概ね良好
- 詳細デューデリジェンスへ進める候補
- ただし投資助言ではなく、現地調査・賃貸需要調査が必要

### A

- 有力候補
- 一部に確認ポイントあり
- 融資条件、修繕履歴、出口価格を確認

### B

- 条件次第
- 指値、自己資金、融資期間、管理費/修繕費次第で検討余地あり

### C

- 慎重検討
- 表面上は成立しても、安全余裕が薄い可能性

### D

- 原則見送り
- CF赤字、DSCR不足、出口難などの重大懸念

### E

- 危険
- 投資対象としては不適合の可能性が高い

## 9. このプロトタイプの限界

- 元サービスの非公開コードは確認していない
- ランク境界はサンプル表示と金融指標の一般的な考え方からの推定
- 立地・土地値・価格妥当性は入力不足だと中立点になる
- 税金、減価償却、譲渡税、法人/個人差、修繕の山は簡略化している
- 正式な投資助言ではない
